# 전략 패턴(Strategy Pattern)
- 특정 작업을 수행하는 여러 알고리즘(전략)들을 각각 클래스로 만들어 캡슐화하고, 필요에 따라 런타임에 동적으로 교체하며 사용할 수 있도록 하는 행위 디자인 패턴입니다. 이를 통해 알고리즘 사용 코드를 직접 수정하지 않고도 다양한 행동을 유연하게 확장할 수 있으며, 복잡한 조건문(if-else)을 제거하고 코드의 재사용성과 유지보수성을 높이는 데 유용합니다.

# 핵심개념

- 알고리즘 군(Family of Algorithms): 동일한 목적을 가지지만 구현 방식이 다른 여러 알고리즘들.

- 캡슐화(Encapsulation): 각 알고리즘을 독립적인 클래스로 만들고, 공통 인터페이스(전략 인터페이스)를 통해 관리.

- 상호 교체 가능(Interchangeable): 컨텍스트(알고리즘을 사용하는 객체)가 실행 중에 어떤 전략을 사용할지 선택하고 변경할 수 있음.
